# 🎫 Customer-Support-Ticket-Analyzer-Using-Python

# Project Overview

**A Python-based text processing and ticket analysis project that automates customer support ticket management, cleans issue descriptions, performs keyword analysis, and generates useful support insights.**


# 📌 Project Overview

Customer support teams receive numerous support requests every day, making it difficult to manually organize, clean, and analyze customer issues.

This project simulates a simple Customer Support Ticket Management System using Python. It demonstrates fundamental data analysis techniques including data cleaning, text preprocessing, keyword analysis, priority analysis, and issue summarization.

The project showcases core Python programming skills and basic text analytics, making it an excellent beginner-level data analytics project.


# 🎯 Project Objectives

- Create a simple customer support ticket management system.
- Store ticket information using Python data structures.
- Allow users to add new support tickets dynamically.
- Clean messy issue descriptions.
- Perform keyword-based issue analysis.
- Analyze ticket priorities.
- Extract useful insights from customer issues.


📂 Dataset

The dataset contains customer support information including:

| Column | Description |
|---------|-------------|
| Ticket_No | Unique ticket number |
| Customer_Name | Name of the customer |
| Issue_Description | Customer issue description |
| Priority | Ticket priority (High, Medium, Low) |

  # ⚙ Features

## 1. Ticket Management

- Displays existing tickets
- Automatically generates Ticket IDs
- Allows users to add new tickets
- Validates ticket priority


## 2. Text Cleaning

The project cleans customer issue descriptions by:

- Converting text to lowercase
- Removing punctuation
- Removing extra spaces
- Standardizing common words
- Producing clean text for analysis


## 3. Keyword Analysis

Counts how many tickets contain specific keywords such as:

- good
- poor
- slow
- excellent

This helps identify recurring customer feedback.


## 4. Priority Analysis

Calculates the total number of:

- High Priority Tickets
- Medium Priority Tickets
- Low Priority Tickets

This helps understand workload distribution.


## 5. Longest Issue Detection

Identifies

- Ticket with the longest issue description
- Customer Name
- Word Count

This helps locate tickets requiring detailed investigation.


## 6. Unique Word Extraction

Extracts all unique words from customer issue descriptions.

This demonstrates basic Natural Language Processing (NLP) preprocessing techniques.


# 🔍 Key Insights Generated

- Number of High, Medium, and Low priority tickets.
- Most frequently occurring keywords.
- Cleaned customer issue descriptions.
- Ticket containing the longest issue description.
- Total number of unique words used by customers.
- Overall ticket summary.
